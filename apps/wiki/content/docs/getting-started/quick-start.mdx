## Send Your First Email

This guide will walk you through sending your first email with Plunk in just a few minutes.

### Step 1: Get Your API Key

1. Sign up or log in to your Plunk dashboard at [app.useplunk.com](https://app.useplunk.com)
2. Create a new project or select an existing one
3. Navigate to **Settings > API Keys**
4. Copy your **Secret Key** (starts with `sk_`)

### Step 2: Send a Transactional Email

Use the `/v1/send` endpoint to send your first email:

```bash
curl -X POST https://api.useplunk.com/v1/send \
  -H "Authorization: Bearer sk_your_secret_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "recipient@example.com",
    "subject": "Hello from Plunk!",
    "body": "This is my first email sent via Plunk.",
    "subscribed": true
  }'
```

### Step 3: Check the Response

If successful, you'll receive a response like:

```json
{
  "success": true,
  "data": {
    "id": "email_id_here",
    "status": "queued"
  }
}
```

The email has been queued and will be sent within seconds!

## Using Variables in Emails

Make your emails dynamic by using variables:

```bash
curl -X POST https://api.useplunk.com/v1/send \
  -H "Authorization: Bearer sk_your_secret_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "user@example.com",
    "subject": "Welcome {{name}}!",
    "body": "Hello {{name}}, welcome to {{company}}!",
    "subscribed": true,
    "data": {
      "name": "John Doe",
      "company": "Acme Inc"
    }
  }'
```

Variables use the `{{variable}}` syntax and are replaced with values from the `data` object.

## Using Templates

Templates allow you to reuse email designs across multiple sends.

### Step 1: Create a Template

In your Plunk dashboard:

1. Go to **Templates**
2. Click **Create Template**
3. Choose **Transactional** type
4. Name your template (e.g., "Welcome Email")
5. Design your email with the visual editor
6. Use `{{variables}}` where you want dynamic content
7. Save the template

### Step 2: Send Using Template

```bash
curl -X POST https://api.useplunk.com/v1/send \
  -H "Authorization: Bearer sk_your_secret_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "user@example.com",
    "template": "welcome-email",
    "subscribed": true,
    "data": {
      "name": "John Doe",
      "company": "Acme Inc"
    }
  }'
```

## Managing Contacts

Before sending marketing emails, you'll want to manage your contacts.

### Create a Contact

```bash
curl -X POST https://api.useplunk.com/contacts \
  -H "Authorization: Bearer sk_your_secret_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "subscribed": true,
    "data": {
      "firstName": "John",
      "lastName": "Doe",
      "company": "Acme Inc"
    }
  }'
```

### List Your Contacts

```bash
curl -X GET "https://api.useplunk.com/contacts?limit=10" \
  -H "Authorization: Bearer sk_your_secret_key_here"
```

## Tracking Events

Track custom events for your contacts to trigger workflows and segment audiences.

```bash
curl -X POST https://api.useplunk.com/v1/track \
  -H "Authorization: Bearer sk_your_secret_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "event": "purchase",
    "data": {
      "product": "Premium Plan",
      "amount": 99.00
    }
  }'
```

This will:
1. Create or update the contact with the email
2. Track the "purchase" event
3. Trigger any workflows listening for this event

## Troubleshooting

### Email not received?

1. Check your spam folder
2. Verify the email address is correct
3. Check your Plunk dashboard for delivery status
4. Ensure your AWS SES is properly configured (if self-hosting)

### Authentication errors?

1. Verify your API key starts with `sk_` (secret key)
2. Check that the key hasn't been regenerated
3. Ensure you're using the correct authorization header format

### Rate limiting?

If you're sending many emails:
- Emails are rate-limited to 14/second (AWS SES default)
- Use the queue system for bulk sends
- Consider creating a campaign for large broadcasts
